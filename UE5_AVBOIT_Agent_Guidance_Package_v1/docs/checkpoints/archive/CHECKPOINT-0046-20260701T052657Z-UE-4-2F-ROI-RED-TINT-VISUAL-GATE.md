# CHECKPOINT-0046 - UE-4.2F ROI Red Tint Visual Gate

UTC Time: 2026-07-01T05:26:57Z

Status: partial

## Summary

UE-4.2F adds a visual acceptance target for the standard `TestSpriteMap1` scene:
inside the user-specified ROI, `PluginAVBOIT_RedTint` must show visibly red
particles, while `EngineDefaultBefore` and `EngineDefaultAfter` remain stable
grey. Evidence must be generated by UE screenshot/readback paths, not desktop
capture.

## Implemented

- Added `Config/AVBOIT/TestSpriteMap1TintROI.json`.
- Fixed `AVBOIT.Niagara.ShowOverlay` to attach to GameViewport or active Editor
  LevelViewport and to request visible, clickable mouse input.
- Extended `AVBOIT.Niagara.Status` with overlay attach target, attach error,
  mouse cursor, and input mode fields.
- Fixed runtime tint material application for plugin modes.
- Added tint-visible fallback draw evidence for UE-4.2F only, while preserving
  blocked-local status for real Niagara draw bridge claims.
- Added `AVBOIT.Niagara.CaptureTintComparison`.
- Extended `Scripts/Run-AVBOITNiagaraSpriteUnlitParity.ps1` with
  `-RunTintVisualGate`, `-TintColor`, and `-TintROI`.
- Added ROI crop, side-by-side, difference, heatmap, coverage mask, annotated
  goal image, metrics, and promotion-decision output.

## Gate

- EngineDefault before/after ROI MAE `<= 1/255`.
- Plugin ROI red coverage ratio `>= 0.98`.
- Mean red advantage `>= 0.12`.
- Raw screenshots: `ScreenshotSource=UE.FScreenshotRequest`,
  `DesktopScreenshotUsed=false`, `OverlayHidden=true`.
- No hard-fail log patterns.

## Notes

Passing this checkpoint only promotes UE-4.2F visual gate evidence. The real
Niagara sprite VF/material/SceneColor bridge remains governed by UE-4.2E hard
gates and may still be `blocked-local`.
