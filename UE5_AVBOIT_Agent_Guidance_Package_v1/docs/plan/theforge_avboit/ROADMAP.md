# The Forge → UE AVBOIT Roadmap

| 阶段 | 主题 | 主要产物 | 状态 |
|---:|---|---|---|
| 0 | 项目与资料基线 | 版本锁、资料清单、Agent 规约 | 已建立 |
| 1 | 恢复 The Forge 1.58 实验场 | 可运行基线、代码地图、现有后端证据 | **当前** |
| 2 | RGB Ground Truth | 三面片、六种提交顺序、误差协议 | 待开始 |
| 3 | AVBOIT 理论与参考实现学习 | 算法契约、PDF 页码索引、Egaku 差异 | 待开始 |
| 4 | The Forge AVBOIT 基础后端 | 新增独立后端、阶段调试证据 | 待开始 |
| 5 | OIT 对比实验 | 图像、稳定性、深度复杂度、性能 | 待开始 |
| 6 | 序列帧沙暴卡片 | 多卡片动态测试与目标内容结论 | 待开始 |
| 7 | 性能与资源冻结 | 质量档位、GPU/显存预算、限制 | 待开始 |
| 8 | UE 迁移契约 | 与 API 无关的数据与阶段契约 | 待开始 |
| 9 | UE 插件学习与播放器 | 独立 Flipbook Player | Deferred |
| 10 | Niagara Adapter | Niagara 作为实例数据生产者 | Deferred |

## 跨阶段门禁

每个阶段内部的每一条用户指令都必须产生 checkpoint；阶段切换还必须有一份明确标记“退出条件是否通过”的 checkpoint。没有归档的工作不能作为进入下一阶段的依据。
